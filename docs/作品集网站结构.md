# 设计师作品集网站结构

## 1. 网站定位

这个网站定位为「作品索引型个人作品集」，不是传统个人品牌长页，也不是自由职业者服务销售页。

核心目标：

- 让面试官快速知道我是谁。
- 让作品成为首页主体。
- 通过分类快速筛选项目。
- 点击作品卡片进入项目详情，查看完整设计过程。
- 用底部 Dock 导航建立类似 App 的轻量浏览体验。

一句话结构：

```text
简单介绍你是谁 -> 直接看作品 -> 按分类筛选 -> 点进项目详情
```

不采用重型首页结构：

```text
Hero -> About -> Skills -> Experience -> Testimonials -> Contact
```

因为这类结构更像自由职业者销售页，容易让首页变重，也会让 Hero、About、Skills 之间出现信息重复。

---

## 2. 页面路由

第一版建议保持页面数量克制。

```text
/
  首页：自我介绍 + 作品分类 + 作品展示

/work/project-a
/work/project-b
/work/project-c
  项目详情页

/resume
  简历页或 PDF 简历入口

/contact
  联系方式，可选，也可以做成弹窗
```

推荐第一版先做：

```text
/
/work/project-a
/work/project-b
/work/project-c
/resume
```

`/contact` 可以暂时不做成独立页面，先通过底部 Dock 触发联系弹窗或复制邮箱。

---

## 3. 首页结构

首页只承担两个任务：

1. 介绍我是谁。
2. 展示我做过什么。

### 3.1 个人介绍区

这个区块等同于首页的 Hero，但不再额外拆出 About。

内容建议：

- 姓名
- 职位定位
- 一句话介绍
- 当前求职方向
- 所在城市
- 简历入口
- 联系入口

示例信息结构：

```text
Amber Xu
UX/UI Designer focused on AI products, mobile experiences and design systems.
Based in Shenzhen / Guangzhou.
Available for Product Designer / UX/UI Designer roles.

[View Resume] [Contact]
```

设计要求：

- 文案短，不写长篇自传。
- 不单独做 About 大区块。
- 不堆 Skills 列表。
- 视觉上可以使用大字号姓名、短介绍和少量状态标签。
- 可以加入轻量粒子、鼠标响应或抽象背景，但不能影响作品入口的可读性。

### 3.2 作品分类 Tab

分类 Tab 是首页核心交互之一，用于快速筛选项目。

建议分类：

```text
All
AI Product
Mobile App
SaaS / Web
Design System
Visual / Brand
```

第一版可以更克制：

```text
All
AI Product
Mobile App
Web / SaaS
Visual
```

交互规则：

- 默认选中 `All`。
- 点击分类后，作品卡片即时筛选。
- 当前分类需要有明确选中态。
- 分类数量控制在 4 到 5 个，避免像后台筛选器一样复杂。

### 3.3 作品展示区

首页作品区是主内容，不是辅助模块。

第一版推荐展示 4 到 6 个项目。

每个作品卡片包含：

```text
项目主图
项目名
一句话说明
项目标签
我的角色
点击进入详情
```

作品卡片示例：

```text
AI Interview Assistant
AI Product / UX Design / Web App
Designed a structured interview-prep workflow for job seekers.
Role: Product Design, UX Flow, UI Design
```

卡片要求：

- 主图要足够大，能体现项目质感。
- 文案要短，避免首页变成项目说明书。
- 标签用于辅助筛选，不要过多。
- 整张卡片可点击进入详情页。

---

## 4. 底部 Dock 导航

底部 Dock 是网站的主导航，参考 Nikola Radeski 网站底部固定胶囊导航的体验。

不做复杂顶部导航。

### 4.1 导航项

建议第一版：

```text
Home
Work
Resume
Contact
```

可选扩展：

```text
Home
Work
Resume
Contact
```

暂时不放 About，因为首页介绍区已经承担了 About 的功能。

### 4.2 交互逻辑

```text
Home
回到首页顶部

Work
滚动到首页作品展示区

Resume
打开 /resume 或下载 PDF 简历

Contact
弹出联系方式，或复制邮箱
```

页面状态：

```text
位于首页顶部        -> Home 高亮
浏览作品分类/列表   -> Work 高亮
进入项目详情页      -> Work 高亮
进入简历页          -> Resume 高亮
打开联系弹窗        -> Contact 高亮
```

### 4.3 桌面与移动适配

桌面端：

- 底部居中悬浮胶囊。
- 黑色或深色背景。
- 圆角较大。
- 每个 Tab 有清晰 hover 和 active 状态。

移动端：

- 底部固定 Tab。
- 保持可点击区域足够大。
- 文案可以保留，也可以图标 + 文案。
- 避免遮挡作品卡片底部内容。

---

## 5. 项目详情页结构

项目详情页负责讲清楚项目深度，不把这些内容塞进首页。

每个项目详情页使用统一模板。

```text
01 项目封面
02 项目背景
03 我的职责
04 设计过程
05 最终方案
06 结果与复盘
```

### 5.1 项目封面

内容：

- 项目名
- 产品类型
- 项目时间
- 我的角色
- 项目关键词
- 主视觉图

示例：

```text
Project Name
AI Product / Mobile App / UX Design
Role: UX/UI Designer
Timeline: 2025
Keywords: AI Workflow, Onboarding, Design System
```

### 5.2 项目背景

回答：

- 这是一个什么产品？
- 面向什么用户？
- 当时要解决什么问题？
- 项目为什么重要？

### 5.3 我的职责

回答：

- 我具体负责哪些部分？
- 和谁协作？
- 我产出了哪些设计内容？
- 哪些决策是我推动的？

### 5.4 设计过程

可包含：

- 需求理解
- 用户/场景分析
- 信息架构
- 核心流程
- 关键页面推导
- 方案对比
- 设计迭代

这个部分不需要把所有过程都放满，重点展示设计判断。

### 5.5 最终方案

可包含：

- 高保真页面
- 核心交互
- 关键状态
- 组件或设计系统
- 多端适配

### 5.6 结果与复盘

可包含：

- 上线结果
- 数据反馈
- 用户或团队反馈
- 项目收获
- 如果重做会优化什么

没有真实数据时，不编造数据，可以写设计结果和团队反馈。

## 6. 视觉与交互方向

整体方向：

```text
克制
高级
作品优先
轻互动
底部 Dock 驱动
```

参考 Nikola Radeski 的不是内容密度，而是：

- 底部固定胶囊导航。
- 大面积安静背景。
- 少量强视觉信号。
- 首页像一个清晰入口，而不是复杂菜单。

适合保留：

- 底部 Dock。
- 强排版。
- 浅灰/黑白/中性色背景。
- 少量强调色。
- 轻微 hover、滚动出现、鼠标响应。

不建议照搬：

- 过多 Services。
- 大量 Testimonials。
- 很长的 Awards 区。
- 过度自由职业者销售页表达。

---

## 7. 第一版范围

第一版只做必要内容。

必须有：

- 首页个人介绍区。
- 作品分类 Tab。
- 4 到 6 个作品卡片。
- 3 个左右项目详情页。
- 底部 Dock 导航。
- 简历入口。
- 联系入口。

暂时不做：

- 独立 About 长页。
- Skills 大模块。
- Testimonials。
- Awards。
- Blog。
- 复杂顶部导航。
- 过重 WebGL 或强 scroll-jacking。

## 9. 一句话结论

这个作品集网站应该做成：

```text
一个带底部 Dock 的高级感作品索引页，加上若干统一模板的项目详情页。
```

首页负责快速识别和作品筛选，项目详情页负责展示设计深度。
