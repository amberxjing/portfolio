# Nikola Radeski 网站拆解：中文解释与翻译

## 先解释：这个网站高级感从哪里来

这个页面的核心不是“元素多”，而是“克制”。它把大部分视觉面积都交给浅灰背景和留白，只用少量黑色、深灰文字和橙色按钮建立重点。这样做的结果是：页面看起来很安静，但不会无聊，因为底部黑色胶囊导航、橙色 CTA、真人照片提供了足够的记忆点。

如果你要借鉴到自己的个人网站，不要先抄动效，先抄它的约束：

- 背景用低对比浅灰，而不是纯白或大面积渐变。
- 主内容不要铺满屏幕，控制成一列约 `500px` 左右的文本宽度。
- 强调色只用于一个关键词、主按钮、少量评分或状态，不要到处都用。
- 导航不要占据顶部大面积，可以做成小而固定的底部控制条。
- 证明材料要比装饰更重要：真人照、客户数、项目数、社交链接，比抽象插画更可信。

一句话说：它的高级感来自“少量强信号 + 大面积安静背景”，不是来自复杂视觉。

# Taste DNA 中文翻译

### 留白负责建立框架
- **触发场景**：当页面需要在一个长单页作品集中展示 About 信息和个人证明时。
- **设计选择**：选择用大面积灰色背景和排版来分组，而不是用盒子、分割线或密集卡片。
- **原因**：读者会感觉这个人对自己的表达很有把握，不需要把每个视口都塞满证明材料。
- **证据**：页面背景是 `#dbdbdb`，主表面是 `#efefef`；About 文本列宽 `532px`；可见内容带约 `1065px`；文本和照片之间约 `232px` 留白；未检测到 CSS grid。

### 强调色像标点符号
- **触发场景**：当页面既需要个性，又需要明确行动入口，但不能让整个页面都变成品牌色块时。
- **设计选择**：只把橙色用于 `Creative` 这个词、星级评价和主 CTA，而不是大面积橙色背景或多种强调色。
- **原因**：一个高饱和颜色如果只在关键处出现，会更像“指令标记”，而不是装饰噪音。
- **证据**：`#e27500` 出现在 `168px x 40px` 主 CTA 和 `Creative` 这个词上；大部分可见文字使用 `#333333` 或 `#6d6d6d`；次级按钮使用 `#eaeaea` 加 `1px solid #333333`。

### 控制很小，存在感很强
- **触发场景**：当一个长单页作品集仍然需要一直可访问的导航时。
- **设计选择**：选择 `528px x 52px` 的底部固定胶囊导航，而不是完整顶部导航栏或大型侧边菜单。
- **原因**：导航一直可达，但用户第一眼读到的仍然是内容，而不是网站框架本身。
- **证据**：固定导航在 `1440px` 视口中位于 `x=449px`、`y=824px`；黑色 `#000000` 外壳；`100px` 圆角；`5px` 内边距；文字 `15px / 21px`；首页控件 `42px x 40px`；联系控件 `118px x 42px`。

### 用真人证明替代抽象装饰
- **触发场景**：当设计师/创始人个人页需要建立可信度时。
- **设计选择**：选择一张真实人物照片，加上客户和评价证明，而不是抽象 3D、重纹理或装饰插画。
- **原因**：读者在看作品集前，需要先把“能力声明”和一个真实的人、真实的外部证明连接起来。
- **证据**：首屏人物照片 `270px x 380px`；`24px` 圆角；阴影 `0px 6px 32px rgba(0, 0, 0, 0.15)`；CTA 下方立即出现 `Trusted by over 130+ clients`；五个社交图标重复为 `39px x 39px`、`10px` 圆角的方形图标。

## 对你个人网站的可执行建议

1. 首屏不要急着放太多作品缩略图，先用一句大标题、一段短简介、一个主 CTA 和一张个人图建立定位。
2. 你的强调色最多只用在一个关键词、一个按钮和极少量状态提示上。
3. 文本列控制在 `500px-560px`，不要让简介横跨整个屏幕。
4. 页面背景可以用 `#dbdbdb` 到 `#efefef` 这一类浅灰层次，避免纯白导致廉价模板感。
5. 如果做底部导航，可以参考 `528px x 52px` 的黑色胶囊，但文案要更适合你：`Home / Work / About / Contact`。
6. 动效重点放在 hover、滚动出现、轻微 opacity/transform，不要一上来做重 3D 或强 scroll-jacking。
